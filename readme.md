## 🚀 快速开始

### 环境要求

- Node.js 18.x 或更高版本
- npm 9.x 或更高版本

### 启动步骤

#### 1. 打开命令提示符

按 `Win + R` 键，输入 `cmd`，回车打开命令提示符窗口。

#### 2. 进入项目目录

```cmd
cd /d D:\opencode learning\ai_resume
```

#### 3. 安装依赖（首次运行）

```cmd
npm install
```

> 如果下载慢，可以使用淘宝镜像源：
> ```cmd
> npm install --registry=https://registry.npmmirror.com
> ```

#### 4. 启动开发服务器

```cmd
npm run dev
```

#### 5. 在浏览器中打开

启动成功后，终端会显示类似以下信息：

```
  VITE v5.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: http://xxx.xxx.xxx.xxx:5173/
```

在浏览器地址栏输入：`http://localhost:5173`

即可访问应用。

#### 6. 使用流程

1. 首页点击「立即制作简历」进入编辑器
2. 点击左侧「AI智能填充」按钮，粘贴个人经历描述
3. 点击「开始识别」，AI自动解析
4. 确认识别结果，点击「确认填充全部」
5. 如需调整，可编辑各模块内容
6. 点击「更换模板」切换样式
7. 点击「导出PDF」下载简历

## 一、项目概述

本项目为一款PC端网页应用，核心为用户提供**在线可视化简历编辑、AI智能识别填充、一键高保真导出PDF**的全流程简历制作服务。产品以低门槛、高专业度为核心，降低用户简历制作的学习成本，通过AI能力提升简历内容质量，最终输出符合职场规范、排版统一的PDF格式个人简历，可直接用于求职投递与打印。

---

## 二、项目核心目标

1. 实现所见即所得的在线简历编辑能力，支持模块化编辑、模板一键切换，全程无复杂排版操作；
2. 集成AI智能识别填充能力，用户粘贴个人经历描述，AI自动解析并填充至各对应模块，解决用户"不会写、写不好"的核心痛点；
3. 实现高保真PDF导出功能，保证网页预览效果与最终导出文件的排版、样式、内容100%一致，适配标准打印与投递需求；
4. 产品轻量化设计，核心功能无强制登录门槛，用户可快速上手完成简历制作，全流程核心操作路径不超过3步。

---

## 三、用户角色与使用场景

### 核心用户角色

| 用户角色 | 权限与核心能力 |
|----------|----------------|
| 未登录用户 | 可使用全部核心功能，包括简历编辑、AI智能填充、模板切换、PDF导出；简历数据实时存储在浏览器本地（localStorage），刷新页面内容不丢失 |

> **说明**: 登录用户功能为本版本不实现内容，见第九章

### 核心使用场景

1. 用户复制/粘贴自己的个人经历描述，通过AI智能识别，一键填充至简历各模块；
2. 用户在编辑器中完善各模块内容，实时预览简历效果；
3. 用户选择喜欢的模板样式，一键切换；
4. 用户完成简历编辑后，导出标准PDF文件，用于线上投递与线下打印。

## 四、整体产品架构

本产品分为四大核心模块，各模块职责清晰，可独立开发与迭代，整体架构如下：

1. **前端页面层**：包含网站入口页、简历编辑器核心页，承载用户所有操作交互；
2. **编辑核心层**：负责简历模块化管理、内容实时渲染、模板样式适配、所见即所得预览；
3. **AI能力层**：负责对接智谱AI（GLM-4-Flash）大模型，实现智能识别填充功能；
4. **导出与存储层**：负责PDF文件生成与导出、用户简历数据的本地存储管理。

## 五、核心功能需求

### （一）网站入口页

1. **功能描述**：作为网站访问入口，向用户传递产品核心价值，提供快速进入编辑器的核心通道，无复杂注册流程。

2. **业务规则**
   - 页面核心CTA按钮为「立即制作简历」，点击后直接跳转至简历编辑器页面；
   - 页面清晰展示产品四大核心亮点：AI智能填充、可视化编辑、多套简历模板、一键导出PDF；
   - 无需强制登录/注册，用户点击核心按钮可直接进入编辑器使用全部核心功能；
   - 页面底部放置隐私政策、用户协议的合规入口。

### （二）可视化简历编辑器核心模块

编辑器采用**左右分栏固定布局**，左侧为编辑操作区，右侧为简历实时预览区，所有编辑操作实时同步至预览区，全程所见即所得。

#### 1. 简历模块化编辑管理

- **功能描述**：将简历内容拆解为独立可编辑模块，支持用户对模块进行内容编辑，适配不同用户的简历需求。

- **业务规则**
  1. 预设6个核心模块，覆盖绝大多数简历场景：基本信息（必填）、教育背景、工作/实习经历、项目经验、专业技能、自我评价；
  2. 每个模块预设对应规范字段：
     - 基本信息：姓名、求职意向、联系电话、邮箱、头像上传、个人主页（可选）
     - 教育背景：时间段、学校、专业、学历、描述
     - 工作/实习经历：时间段、公司名称、岗位名称、经历详情
     - 项目经验：时间段、项目名称、项目角色、项目描述
     - 专业技能：技能名称、熟练度
     - 自我评价：自我评价文本
  3. 所有模块的编辑操作，实时同步至右侧预览区，无延迟、无卡顿，实现所见即所得。

#### 2. 内容编辑基础能力

- **功能描述**：为用户提供规范、易用的文本编辑能力，既满足简历内容编辑需求，又避免复杂操作导致的排版错乱。

- **业务规则**
  1. 文本编辑采用普通文本框，不开放复杂排版权限，保证简历格式规范统一；
  2. 经历类模块（工作/实习、项目经验、教育背景）支持多条条目新增、删除，每条条目可独立编辑，适配多段经历的展示需求；
  3. 头像上传支持本地图片上传，采用FileReader读取为Base64格式存储；
  4. 所有输入内容支持实时自动暂存（localStorage），刷新、关闭页面后重新打开，可恢复最近一次的编辑内容。

### （三）AI智能填充核心模块

本模块为产品核心亮点，通过AI智能识别实现一键填充。

#### 1. AI智能识别填充

- **功能描述**：用户粘贴大段个人经历描述，AI自动解析并填充至各对应模块。

- **业务规则**
  1. 功能入口设置在编辑器左侧栏上方，醒目的紫色渐变按钮「AI智能填充」；
  2. 用户点击按钮后弹出大文本输入框，支持粘贴大段个人经历描述；
  3. AI解析文本，提取以下信息并对应填充：
     - 基本信息：姓名、电话、邮箱、求职意向、个人主页
     - 教育背景：时间段、学校、专业、学历、描述
     - 工作经历：时间段、公司名、岗位、描述
     - 项目经验：时间段、项目名、角色、描述
     - 专业技能：技能名称
     - 自我评价：总结性描述
  4. 识别完成后显示识别结果预览，用户确认后一键填充全部模块；
  5. 填充前自动清空当前简历全部内容，初始化界面后再填充AI识别结果；
  6. 采用智谱AI GLM-4-Flash模型进行文本理解与信息提取。

### （四）简历模板管理模块

1. **功能描述**：提供多套预设的标准化简历模板，用户可一键切换，所有简历内容自动适配新模板的排版规范。

2. **业务规则**
   - 预设3套不同风格的简历模板：
     - 简约商务风：简洁大方，适合传统行业求职
     - 校招应届生风：清新蓝调，适合校招场景
     - 互联网创意风：活力紫色，适合互联网行业
   - 模板切换为全局生效，切换后所有已编辑的简历内容，自动适配新模板的字体、间距、配色、排版规范；
   - 所有模板严格适配A4标准纸张尺寸；
   - 模板选择入口设置在编辑器页面顶部，支持用户点击预览模板效果，确认后一键应用。

### （五）PDF导出功能模块

1. **功能描述**：支持用户将编辑完成的简历，一键导出为高保真、可打印、适配投递规范的PDF格式文件。

2. **业务规则**
   - 导出入口固定在编辑器页面顶部，按钮命名为「导出PDF」；
   - 导出文件采用A4标准纸张尺寸；
   - 导出的PDF文件与编辑器右侧预览区的内容、排版、样式100%一致；
   - 导出文件默认命名规则为「姓名-求职意向-个人简历.pdf」；
   - 导出过程给出加载状态提示，导出完成后自动触发浏览器下载。

### （六）数据存储与管理模块

1. **功能描述**：实现用户简历数据的安全、稳定存储，保障用户编辑内容不丢失。

2. **业务规则**
   - 采用浏览器本地存储方案（localStorage + Zustand persist）；
   - 实时自动保存用户的所有编辑内容；
   - 页面刷新、关闭后重新打开，可恢复最近一次的编辑内容；
   - 提供「清空内容」功能，一键重置简历为初始空白状态（需二次确认）。

### （七）通用基础功能

1. 页面适配：支持PC端主流浏览器（Chrome、Edge、Firefox、Safari）；
2. 重置功能：支持用户一键清空当前所有编辑内容，操作时需二次确认；
3. 合规功能：页面底部放置隐私政策、用户协议入口。

## 六、非功能需求

### （一）性能需求

1. 页面加载速度：首页首屏加载时间不超过2秒，编辑器页面首次加载时间不超过3秒；
2. 编辑响应速度：所有编辑操作实时同步至预览区，延迟不超过100ms；
3. AI响应速度：AI智能填充请求的平均响应时间不超过10秒；
4. 导出速度：单页简历导出时间不超过3秒。

### （二）兼容性需求

1. 浏览器兼容：支持Chrome 80+、Edge 80+、Firefox 75+、Safari 14+ 等主流PC浏览器；
2. 设备兼容：支持1366*768及以上分辨率的PC设备；
3. 文件兼容：导出的PDF文件可在所有主流PDF阅读器中正常打开、打印。

### （三）安全与合规需求

1. 数据安全：用户本地存储的数据仅保存在用户浏览器端，不上传至服务器；
2. 内容合规：AI生成的内容需符合相关法律法规；
3. 合规要求：网站整体符合《网络安全法》《个人信息保护法》相关要求。

### （四）可用性需求

1. 核心功能操作路径不超过3步；
2. 所有按钮、操作入口有明确的语义提示；
3. 操作异常、请求失败时，给出友好、明确的错误提示；
4. 页面整体UI风格统一，符合现代网页设计规范。

## 九、需求边界（本版本不实现）

1. 暂不支持移动端页面适配，仅聚焦PC端网页开发；
2. 暂不支持拖拽排序功能；
3. 暂不支持模块隐藏/显示功能；
4. 暂不支持自定义模块功能；
5. 暂不支持复杂的富文本排版功能；
6. 暂不支持登录/云端存储功能；
7. 暂不支持AI生成、AI润色、AI岗位匹配功能（仅保留智能填充）；
8. 暂不支持多人协作编辑功能。
